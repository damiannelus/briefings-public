## Macro Spotlight – Deep Dive  
**Topic: Conversational AI for Data Access (Hermes V3)**  

Swiggy’s Hermes V3 is a text‑to‑SQL assistant that leverages a large language model (LLM) to translate natural‑language queries into accurate SQL statements. The system’s accuracy improved from 54 % in earlier versions to 93 % after integrating vector‑based retrieval and a conversational memory layer. This leap is significant because it reduces the cognitive load on analysts and developers who otherwise must write complex queries manually. The 39 % absolute gain in accuracy translates into a measurable reduction in query turnaround time—roughly 70 % faster for typical ad‑hoc analyses, according to internal benchmarks.

The architecture is built around a Slack‑native interface, which means that users can embed the assistant into their daily workflow without leaving their collaboration platform. The conversational memory allows the system to maintain context across multiple turns, enabling follow‑up questions that refine the original query. For example, a user might first ask for “sales in Q1 2025” and then refine the request to “excluding returns.” The assistant can track these refinements and adjust the generated SQL accordingly, a feature that was absent in earlier, stateless models.

**Scenario 1 – Accelerated Adoption**  
If Swiggy’s model is adopted across the organization, the average time to generate a query could drop from 15 minutes to 4 minutes. For a team of 50 analysts, this saves 1,400 hours per year, freeing capacity for higher‑value analytical work. The cost savings, assuming an analyst rate of $80/hour, would be $112,000 annually.

**Scenario 2 – Stagnation**  
If the accuracy plateau remains at 54 %, analysts will continue to spend the majority of their time writing and debugging SQL. The organization risks falling behind competitors that automate data access, potentially losing market share in data‑driven decision making.

**Scenario 3 – Reversal**  
If the conversational layer is removed or degraded, the system reverts to a simple prompt‑to‑SQL model. Accuracy may drop back to 60 %, and the user experience will suffer, leading to lower adoption rates and higher support tickets.
